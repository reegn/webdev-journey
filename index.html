Heading Element

This is the heading element. It is made with an opening tag and a closing tag.

Opening Tag:

Opening tag is made with angle brackets '<' and '>' and denote the start of an html element. The tag name goes inside.

Closing Tag:

Closing tag is similar to opening tag, and denotes the end of an html element. The tag name consists of a forward slash / followed by the tag name.

Content:

Content goes in between the opening and closing tags. They are what is shown on the website itself.

Element:

The html element is the entirety of the opening and closing tags in addition to the content inside.

The Heading Element:

The heading element is usually used for section headers like a table of contents. The heading element can only go up to h6, starting from h1.

Acceptable heading implementation:

There should usually only be one h1 header element, and no skipped heading levels.

Void Elements

Void elements are elements that do not include content. An example would be a blank line, or a straight line. The horizontal rule element and the break rule element are examples. Their tags are also known as self closing tags, since their tags close themselves. Hence, they are made with only one tag.

Horizontal Rule Element:

The horizontal element is made with angle brackets '<' and '>' with the tag name "hr /" and is the straight line seperating text:

This is slightly different from the closing tags, where the forward slash is right after the tag name, rather than before it. "/p" for the paragraph element's closing tag vs. "hr /" inside the brackets.

Break Element:

The break element is made similar to the horizontal rule element,
with its tag name being "br /" and adds a new empty line for the text to be on (similar to pressing enter).
They can be used to break lines within paragraph elements.
Useful for making poems with it, allowing for formatting seperate lines of text.
Rather than writing poems as large paragraphs.

Acceptable Implementations for these Elements:

Create seperate paragraph elements rather than using the break element to seperate lines within paragraphs. Only use it when there is a reason to seperate different lines in a paragraph and keeping them as a single paragraph. In other words, do not use the break element to format a paragraph element to make the paragraph itself, such as adding a break element to each line in a paragraph to format it in a certain shape. Only do so for a reason.

Valid Tag Name:

It is valid to write both the break and horizontal rule elements without the forward slash. "br" instead of "br /" as the tag name within the brackets acceptable for the break rule. It is the same case for the horizontal rule.
